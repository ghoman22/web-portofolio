How I Built a CI/CD Pipeline Using GitHub Actions

Firman Arya DevOps Engineer
December 15, 2024
8 min read
1.2k views
GitHub Actions Docker Kubernetes DevOps

In today's fast-paced development environment, having an efficient CI/CD pipeline is crucial for delivering software quickly and reliably. In this post, I'll walk you through how I designed and implemented a robust CI/CD pipeline using GitHub Actions that reduced our deployment time from days to minutes.

01 The Challenge

Our development team was facing several challenges with our manual deployment process:

Slow Deployments

Manual deployments taking 2-3 days due to complex coordination

Human Errors

Frequent mistakes during the release process

Environment Inconsistency

Lack of consistency between different environments

No Testing

No automated testing before production deployments

Solution Goal

We needed a solution that would automate our entire deployment pipeline while maintaining high quality and security standards.

02 Architecture Overview

The CI/CD pipeline I designed follows a multi-stage approach with clear separation of concerns:

Source Control

GitHub Repository

Build & Package

Docker Container

Test & Validate

Automated Testing

Deploy

Kubernetes Cluster

03 GitHub Actions Workflow

Here's the complete GitHub Actions workflow that powers our CI/CD pipeline:

YAML .github/workflows/ci-cd.yml 
name: CI/CD Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests
        run: npm run test:coverage
      
      - name: Upload coverage reports
        uses: codecov/codecov-action@v3

  build:
    needs: test
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    
    steps:
      - name: Checkout repository
        uses: actions/checkout@v3
      
      - name: Log in to Container Registry
        uses: docker/login-action@v2
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      
      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v4
        with:
          images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}
          tags: |
            type=ref,event=branch
            type=ref,event=pr
            type=sha,prefix={{branch}}-
      
      - name: Build and push Docker image
        uses: docker/build-push-action@v4
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}

  deploy:
    needs: build
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    
    steps:
      - name: Deploy to Kubernetes
        uses: azure/k8s-deploy@v1
        with:
          manifests: |
            k8s/deployment.yaml
            k8s/service.yaml
          images: |
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:main-${{ github.sha }}

06 Lessons Learned

Start Simple, Iterate

Don't try to build the perfect pipeline from day one. Begin with basic automation and gradually add complexity.

Security First

Build security into the pipeline from the beginning, not as an afterthought. Use secrets management and vulnerability scanning.

Monitor Everything

You can't improve what you don't measure. Implement comprehensive monitoring and alerting from the start.

Documentation Matters

Clear documentation is crucial for team adoption, troubleshooting, and long-term maintenance.
